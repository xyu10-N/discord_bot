# Discord Bot

このリポジトリは、Discordサーバーの運用を補助するための各種自動化機能を持ったカスタムDiscord Botです。

## 機能概要

### 1. 定期送信・アンケート機能 (`/hona` スラッシュコマンド)
特定のチャンネルに対して、定期的なアナウンスとアンケートを行います。

- **開始コマンド:** `/hona start`
- **停止コマンド:** `/hona stop`
- **スケジュール内容:**
  - **毎時42分:** 「次鯖」に関するアンケート（あり/なし/ココ）をボタン付きで送信します。
  - **毎時50分:** 上記アンケートの回答結果を集計し、メンション付きで送信します。
  - **毎時58分:** 時速確認のリマインドを送信します。
  - **2時間半ごと:** 「炊いてくださーい！」のメッセージを自動送信します。

### 2. 部屋番号監視とチャンネル名変更
指定されたチャンネル（`config.js`にて複数設定可能）で「5桁」または「6桁」の数字のみのメッセージが送信されると、特定のチャンネル名を自動的に `部屋番号【〇〇】` に変更します。

### 3. キーワード自動返信
メッセージ内に特定のキーワード（部分一致）が含まれていた場合、あらかじめ設定されたテキストを返信します。

### 4. キーワード自動リアクション
メッセージ内に特定のキーワード（部分一致）が含まれていた場合、指定されたカスタム絵文字で自動的にリアクションを付与します。

## 環境構築と起動方法

本Botは Docker を用いて簡単に起動・運用が可能です。

### 前提条件
- Docker および Docker Compose がインストールされていること
- Discord Developer Portal にて Bot を作成し、**Message Content Intent** が有効になっていること

### 1. 環境変数の設定
プロジェクトのルートディレクトリに `.env` ファイルを作成し、DiscordのBotトークンを設定してください。

```env
DISCORD_TOKEN=あなたの_BOT_TOKEN_をここに記載
```

### 2. 設定のカスタマイズ
チャンネルIDや絵文字ID、キーワードなどはすべて `config.js` で管理されています。ご自身のサーバーに合わせて `config.js` の値を適宜変更してください。

### 3. Dockerでの起動

以下のコマンドを実行することで、イメージのビルドとバックグラウンドでのコンテナ起動が行われます。

```bash
docker compose up -d --build
```

**ログの確認方法:**
正常に起動しているか、またはエラーが出ていないかを確認するには以下のコマンドを使用します。

```bash
docker compose logs -f
```

**Botの停止方法:**

```bash
docker compose down
```
